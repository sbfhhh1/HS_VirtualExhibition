# HS_VirtualExhibition 问题排查与修复总结（2026-06-24）

本文记录本次会话解决的几个问题、根因与最终修复，并保留踩过的坑作为后续参考。

---

## 1. 官方 UE5.8 MCP 接入

- 项目用的是 **Epic 官方 `ModelContextProtocol` 插件**（引擎内置，Streamable HTTP，`http://127.0.0.1:8000/mcp`），不是第三方 Python 适配器。
- 已在 Claude Code 注册官方 connector `unreal-mcp`（local 作用域）。
- 工具集：默认开启 Tool Search，启动时只暴露 `list_toolsets / describe_toolset / call_tool` 三个入口工具，其余按需发现。
- **服务器默认不自动启动**：需要时在 UE 控制台执行 `ModelContextProtocol.StartServer`。

## 2. 打包后「抓取手势失效」修复（已验证通过）

打包版抓取手势无法触发印章与背景轮回（编辑器/Standalone 正常），三处根因：

1. **背景不轮回** — BG 背景板只有编辑器 label、无真 Tag，`#if WITH_EDITOR` 在打包版被编译掉 → 给 `StaticMeshActor_1` 加真 Tag `BG`。
2. **印章不显示** — 程序网格源文件 `SourceArt/ChineseSeals/*.sealmesh` 没进包。修复：`Config/DefaultGame.ini` 加
   `+DirectoriesToAlwaysStageAsNonUFS=(Path="../SourceArt/ChineseSeals")`
   - 必须 **NonUFS**（本项目 ZenStore/IoStore 打包，UFS 文件进容器后运行时 `FPaths::ProjectDir()` 读不到）。
   - 路径必须 `../` 开头（`DirectoriesToAlwaysStage*` 相对 `Content/` 解析，文件在项目根）。
3. **代码硬化** — `FistDisplayToggleComponent` / `BurstMeshStateSwitcherComponent` 去掉对 `GetActorLabel()` 的运行时依赖，统一用真 Tag / 名称匹配，杜绝「编辑器好、打包坏」。

> 教训：运行时逻辑别用 `GetActorLabel()`；唯一可靠的验证是真打包（Standalone 仍带 WITH_EDITOR，骗不出问题）。

## 3. MCP 自动启动 vs 打包冲突

- 官方 MCP 的 `bAutoStartServer` **必须保持 False**。
- 否则从编辑器打包时，cook 子进程会去抢 8000 端口（与正在运行的编辑器冲突）→ `LogHttpListener: Error` → cook 返回非零 → 打包失败。
- 配置位置：`Config/DefaultEditorPerProjectUserSettings.ini` 的 `[/Script/ModelContextProtocolEngine.ModelContextProtocolSettings]`。

## 4. 玉石 Substrate SSS「透光感」缺失 —— 真因是**场景灯光**

现象：玉石「C」(中华玉龙 ZHYL，材质 `MI_ZHYL_Jade_SSS`) 在编辑器/PIE/打包里都没有 SSS 透光感，看起来像不透光的金黄色；但**材质编辑器预览球有 SSS**。

### 最终结论
**材质与 Substrate 配置一直都是对的，问题是场景灯光不足。** 「C」嵌在昏暗的木质内龛里、主要靠 Lumen 间接光，光太弱，SSS 透射（透光感）出不来。**给玉石补强灯光后，透光玉石感立即出现。**

### 判定依据（关键）
- 预览球有 SSS、场景没有，而两者共用同一套全局渲染设置 → **排除全局配置**（配置错会让预览也没 SSS）。
- 因此是场景特定问题 → 灯光。

### 排查中确认过的事实（这些都不是真因）
- `r.Substrate=True`、`r.SubsurfaceScattering=1`、`r.SSS.Quality=1` 运行时均生效。
- 引擎不存在 `r.Substrate.Subsurface` 这个 cvar。
- 材质 `M_YZL_ProceduralJade_SSS` 是 `MSM_SUBSURFACE`，Substrate 开启时自动转换，主节点有 `Front Material` 针脚 → Substrate 确实生效。
- 场景「C」用的就是 `MI_ZHYL_Jade_SSS`（与预览同一材质）。

### 当前渲染配置（`Config/DefaultEngine.ini`）
- `r.Substrate=True`
- `r.Substrate.ProjectGBufferFormat=1`（Adaptive GBuffer，需 SM6，项目已开）
  - 注：排查中一度怀疑此项并来回改过，最终证明与本问题无关；保留为 1（Adaptive，功能更全）。
- 同时写入 `[SystemSettings]` 段做锚定（编辑器退出重写 `[RendererSettings]` 时不会冲掉它）。

> 教训：「预览有、场景无」「加灯就出现」是**灯光问题**的典型特征。下次优先排查灯光，不要一头扎进渲染配置。
